# 推しコミリマインダー

Web漫画の更新日をLINEで受け取れる通知機能と、自身の購読リストをWeb上で公開・共有できる機能を備えたサービスです。

## 1. サービス概要
- **どんなサービスか**: 散在するWeb漫画の更新情報をLINEに集約する通知機能と、自身の「購読リスト」をWeb上で公開・共有できる機能を備えたリマインド＆布教ができるWebアプリ。
- **誰の、どんな課題を、どう解決するか**: 複数のマンガアプリを掛け持ちしているユーザーの「更新日を忘れて、無料公開期間を逃してしまう」「各アプリを巡回するのが面倒」という課題を、使い慣れたLINEへの自動通知で解決します。さらに、自身の購読状況をシェアできる仕組みを提供することで、ファン同士の「布教」と「新たな作品との出会い」を促進します。

---

## 2. このアイデアはどこから生まれたか
### 2-1. きっかけとなった体験・感情
複数のWebマンガアプリを愛読していますが、週刊、隔週、月1更新など更新日がバラバラで更新日を忘れてしまうことがよくあります。公開から1週間は無料で読める作品も多いのですが、気づいた時には有料期間に入っていて「読み逃した！」と落胆することもしばしば。楽しみにしているからこそ、その「落胆」や「読み逃し」をなくしたいと感じたのがきっかけです。
また、SNSのハッシュタグなどで流れてくるおすすめ漫画の布教ポストを見て、面白い漫画に出会えるのもマンガ好きとしての楽しみの一つです。
私にとって、リアルタイムで連載を追いかけることは「明日も頑張ろう」と思える大きな生きる活力になっています。この「最新話を追いかける楽しさ」を自分だけのメモで終わらせず、他の人と共有したり、新しい漫画を布教し合える場所があったら面白そうだなと感じました。

### 2-2. なぜそれが「気になった」のか
自分は現在Webエンジニアを目指して学習中であり、日常の「不便」を「仕組み」で解決することに興味があるからです。また、多忙な毎日の中で、趣味の漫画を読む時間は貴重な癒やしであり、その時間をより確実に、ストレスなく確保したいという思いがあります。

## 3. 課題の整理
### 3-1. 表に見えている困りごと
- 漫画の更新日を忘れて、SNSで先に衝撃展開のネタバレを知ってしまい楽しみが奪われた。
- 更新を確認するために何度もブラウザやアプリを開き直すのが面倒で、漫画を追うのが楽しみではなく義務的な作業に感じてしまう。
- Web漫画の中には最新話公開から1週間など限定無料で配信されることも多い。更新日を失念し、数日の差で本来無料で読めたはずのものが有料（課金対象）に切り替わってしまうことで、金銭的な損だけでなく「自分の不注意で読み逃した」という自己嫌悪に繋がる。
- 一度更新を逃して話数が空いてしまうと、「1話読み逃したからもういいや」という心理的ハードルが生じ、そのまま作品を追うのをやめてしまうきっかけになる。

### 3-2. 本当に解決したい課題は何か
- なぜ？：楽しみな体験（漫画を読むこと）のタイミングを逃したくないから。
- なぜ？：自分の好きなコンテンツを主体的に追いかけたいから。
- なぜ？：日常のタスクに追われても、自分の「好き」を忘れない心の余裕を持ちたいから。
- **本当に解決したい課題**: 「自分の好きなもの」へのアクセスを便利にし、忙しい日々でも楽しみを忘れないこと。

## 4. 想定ユーザーについて
### 4-1. 想定しているユーザー
- **ペルソナ**: 
  **人物像**:20代後半女性、社会人。5つのマンガアプリに加え、ブラウザ（Webサイト）のみで連載されている作品も並行して購読している。
- **具体的な悩み**: 週刊・隔週・月刊と更新頻度がバラバラで、「今日が何の更新日か」を把握するために各アプリやサイトを巡回するのが日課になっている。
  **失敗体験**: 楽しみにしていた作品の無料公開期間（1週間限定など）をうっかり失念。気づいた時には有料チケットが必要な期間に切り替わっており、「読み忘れてしまった」という後悔を味わった経験がある。

### 4-2. 自分とユーザーの距離
- **自分自身**: 私自身がまさにこの課題を感じており、自分が「本当に毎日使いたい」と思えるものを形にしたい。またSNSの布教ポストで面白い作品を知った経験があることから、リマインドアプリとかけ合わせてみたら面白そうだと思ったから。

### 4-3. 実際に使われる可能性について
- **まず最初に使ってくれそうな人は誰か？**: 同じWebエンジニアスクールの仲間。
- **その人は、どんな場面でこのアプリを思い出すか？**: 好きな作品の最新話が公開される直前や、新連載を読み始めた時。
- **今の生活の中で、代わりに使っているものは何か？**: 各Web漫画アプリのプッシュ通知、作品公式Xの投稿、リマインダーアプリ。
- **それを置き換えてまで使う理由はあるか？**: 
    - Web漫画アプリは作品の更新までリマインドしてくれる機能がない。
    - 公式Xの通知をONにすると、宣伝やリポストなど更新以外の全投稿も届くため、見逃しが発生しがち。また、マイナー作品には公式アカウント自体が存在しないケースも多い。
    - リマインダーアプリは「自分用のメモ」で完結するが、本アプリは、リマインド登録というアクションをそのまま「購読リストのシェア」へと繋げ、リアルタイムで連載を追いかける楽しさを共有できる。

### 4-4. 導入・継続の流れ
- **きっかけ**: 新しい連載を読み始めた時に「これの通知が欲しい」と思い出して利用を開始。
- **最初の一歩**: LINE連携を行い、漫画のタイトルと頻度（例：毎週月曜）を登録する。
- **利用を続ける理由**: 
  - 一度設定すれば、あとは何もしなくてもLINEに通知が届くという「完全自動の体験」が便利だから。
  - 他の人の購読リストを見ることで面白い作品の発見に繋がるから。

### 5-2. それでも自分が作りたい理由
- Web漫画アンテナ
実際に利用していたが、自分からサイトにアクセスしないと更新が確認できないので、結局見に行くのを忘れてしまうことが多々あった。また、個人連載などのマイナーな漫画はサイト側に登録されていないこともあり、自分の追いたい作品を網羅しきれなかった。

- ブックライブのみんなの公開リスト
完結済みの作品がリスト化されていることが多く、連載中の作品に特化したリストは少ない。今まさに連載を追いかけているワクワク感を他の人と共有できれば。

- ジャンプ+、サンデーうぇぶり、マガポケなどのマンガアプリ
アプリの通知をONにしても、キャンペーン告知や新作品の宣伝ばかりが届き、お気に入り作品の更新をピンポイントで教えてくれる機能がない。

- ToDo リスト（Googleのリマインダーアプリ）
リマインド機能だけならリマインダーアプリで十分だが、購読リストをシェアできるという付加価値をつけることで、リマインダーとしての実用性と新しい作品に出会える楽しさを両立させたい。

### 5-3. 差別化を一文で決める
- **私のアプリの差別化ポイント**: 使い慣れたLINEでお気に入り作品の読み逃しを防止するだけでなく、お気に入り作品を布教することでリアルタイムで連載を追いかけるわくわく感をシェアできる

## 7. このアプリで実現すること
### MVPで作る機能
- LINEログインによるユーザー管理: ログイン時に通知用のLINE内部ユーザーIDを取得・保持します。
- 漫画タイトル、更新周期の登録・編集・削除機能
- リマインダー登録時の利便性を高めるため、楽天ブックス書籍検索APIによるマンガタイトルの自動入力を実装します。なお、検索でヒットしない作品がある可能性を考慮し、自由入力機能も併設します。
- LINEプッシュ通知機能: 設定された更新日に基づき、自動でLINEメッセージを送信します。
- 非同期処理・自動リトライ: 送信失敗時に備え、Sidekiqを用いた通知の仕組みを構築します。
- 推しコミリストの公開機能: 自分がリマインド登録している作品を「推しコミリスト」として、Webアプリ上で公開できます。イメージは、リマインダーリスト一覧ページで「推しコミリストを作成する」ボタンを押すと、作品名が自動で入力された推しコミリストが作成されます。
作品ごとにコメントを添えられて、推しコミリストにタイトルをつけて公開できます。リストを見た人が思わず読みたくなるような布教の場を提供します。
推しコミリストは公開/非公開が選択可能です。

### ユーザーが入力する情報
- LINEログインの Scopes（スコープ） を利用して、ユーザーの同意を得た上で以下の情報を取得します。
  - openid
  - profile
    - ユーザーID: ユーザーIDを利用してメッセージを送信する
    - 表示名 / プロフィール画像: アプリ内での表示用
- ユーザーがリマインダーに入力できる情報
  - 作品名（必須）
  - 通知日時（必須）
  - 作品URLなど（任意）
- ユーザーが推しコミリストに入力できる情報
  - リストのタイトル（必須）
  - 作品名（必須）: リマインダーから引用
  - 一言コメント（任意）
  - 画像（任意）
  - URL（任意）

### 本リリースで作る機能
- LIFF (LINE Front-end Framework)の実装: LINEアプリ内から設定画面をシームレスに開けるようにします。

## 8. このアプリの懸念点とその対策
### 懸念点①
- **何が問題になりそうか**: 漫画の連載終了や休載、更新日の変更に自動で対応できない。
- **なぜそう思うか**: ユーザーが自分で情報を更新し続けるのは面倒だから。
- **そのために考えている対策**: LINE通知内に「設定を変更する」ボタンを設置し、通知を受け取ったついでに変更できる導線を作る。

### 懸念点②
- **何が問題になりそうか**: リスト作成から年数が経つと連載が終了している作品が出てくる。
- **なぜそう思うか**: リアルタイムで連載を追っている漫画を共有できることを重視しているため。
- **そのために考えている対策**: 
  - リストの作成日から判別してリストの年別カテゴリーを設けることで、「今連載されている作品なのか」を一目で分かるようにする。
  - リストに「作成日」を記載することで、「古い情報かもしれない」という前提でリストを見てもらえるため、連載終了が混ざっていてもネガティブな印象を与えにくい。

### 本サービスの概要（700文字以内）
本サービスは、複数のマンガアプリを併用する読者が抱える「更新日を忘れて、無料公開期間を逃してしまう」「各アプリを巡回するのが面倒」という課題を解決するためのリマインド＆布教アプリです。

【想定ユーザー】
20代を中心とした社会人。複数のアプリやWebサイトでマンガを購読しており、各アプリを巡回して更新を確認することに手間を感じている層をターゲットとしています。

【アプリの目的】
使い慣れたLINEへの自動通知によって読み逃しのストレスを解消し、同時に自身の購読リストをシェアできる仕組みを提供することで、ファン同士の布教と新たな作品との出会いを促進することを目的としています。

【主な機能】
LINEログインにより、ユーザーはリマインダーを登録するだけで、更新日にLINEでプッシュ通知を受け取ることができます。また、登録した作品を推しコミリストとしてWeb上に公開でき、一言コメントを添えて他のユーザーへお気に入り作品を布教することが可能です。

### MVPで実装する予定の機能
- LINEログイン機能
  Devise × OmniAuth を用いてユーザー認証を行う
　参考URL:https://qiita.com/tkhero555/items/35a6330725c9e673ccdc
- リマインダー管理機能
  マンガタイトル・通知日時の登録・編集・削除
- 楽天ブックス書籍検索APIによるマンガタイトル自動入力機能
  リマインダー登録時に、楽天ブックス書籍検索APIによるマンガタイトルの自動入力を行う
  マンガのサムネイルは楽天ブックス書籍検索APIから画像URLを参照して表示させる
- LINEプッシュ通知機能
  設定日に基づく自動通知、Sidekiqによる非同期処理
- 推しコミリスト作成・公開・編集・削除機能
  リマインダーからのマンガリストを自動引用、リストのタイトル・一言コメント・画像の登録
- 推しコミリスト閲覧機能
  未ログインユーザーを含む他者のリスト閲覧、リストを公開/非公開に選択することも可

### Webアプリの概要
本サービスは、複数のマンガアプリを併用する読者が抱える「更新日を忘れて、無料公開期間を逃してしまう」「各アプリを巡回するのが面倒」という課題を解決するためのリマインド＆布教アプリです。

【想定ユーザー】
20代を中心とした社会人。複数のアプリやWebサイトでマンガを購読しており、各アプリを巡回して更新を確認することに手間を感じている層をターゲットとしています。

【アプリの目的】
使い慣れたLINEへの自動通知によって読み逃しのストレスを解消し、同時に自身の購読リストをシェアできる仕組みを提供することで、ファン同士の布教と新たな作品との出会いを促進することを目的としています。

【解決したい課題】
複数のマンガアプリを掛け持ちしているユーザーが抱える「更新日を忘れて無料公開期間を逃してしまう」「各アプリを巡回するのが面倒で漫画を追うことが義務的な作業になってしまう」という課題を、使い慣れたLINEへの自動通知で解決します。さらに、自身の購読リストをシェアできる仕組みを提供することで、忙しい日々の中でも「自分の好きなもの」を忘れず、ファン同士の布教と新たな作品との出会いを促進します。
